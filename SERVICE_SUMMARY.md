# AI 입학사정관 — Product Requirements Document (PRD)

---

## 1. 제품 개요

| 항목 | 내용 |
|------|------|
| **제품명** | AI 입학사정관 |
| **슬로건** | "합격을 설계하는 AI" |
| **제품 유형** | B2B SaaS 웹 애플리케이션 |
| **타겟 시장** | 대한민국 고등학교 (약 2,400개교) |
| **핵심 가치** | 2028 대입 개편에 대응하여, 공공데이터 API와 AI를 활용해 학생의 대입 전략 수립 → 생활기록부 작성 → 서논술형 채점까지 교사의 전체 입시 지도 워크플로우를 하나의 플랫폼에서 지원 |

## 2. 배경 및 문제 정의

### 2.1 시장 배경
- **2028 대입 개편**: 교과 이수 경로가 다양화되면서 교사가 학생별 맞춤 입시 전략을 수립해야 하는 부담 증가
- **생기부 작성 부담**: 세특(세부능력 및 특기사항), 창체(창의적 체험활동), 행발(행동특성 및 종합의견) 등 학생별 1,500 Byte 내외의 기록을 교사가 수작업으로 작성
- **서논술형 평가 확대**: 객관식에서 서술형으로 평가 방식이 전환되면서 공정하고 일관된 채점 기준 적용의 어려움

### 2.2 핵심 문제
1. **정보 비대칭**: 197개 대학의 입결 데이터가 분산되어 있어 교사가 학생별 합격 가능성을 파악하기 어려움
2. **반복 업무 과중**: 32명 기준 학급 단위로 세특·창체·행발을 개별 작성하는 데 수십 시간 소요
3. **품질 리스크**: 학생 간 세특 유사도, 금지어 포함, 바이트 초과 등 NEIS 입력 오류 발생 가능성
4. **채점 일관성**: 서논술형 답안의 주관적 채점으로 인한 공정성 이슈

## 3. 타겟 사용자

### 3.1 주요 페르소나

| 구분 | 설명 |
|------|------|
| **1차 사용자** | 고등학교 교과 담당 교사 (예: 물리I 담당 김영희 선생님) |
| **역할** | 교과 수업 + 학생 생기부 작성 + 서논술형 채점 + 입시 상담 |
| **Pain Point** | 학급당 30~35명의 세특을 개별 작성하는 시간적 부담, 입결 데이터 수집·분석의 어려움 |
| **목표** | 생기부 품질 향상 + 채점 공정성 확보 + 학생별 맞춤 입시 전략 수립 |

### 3.2 사용 시나리오
- **학기 중**: 수업 관찰 메모 기록 → AI 세특 자동 합성 → 유사도/금지어 검증
- **시험 후**: 서논술형 답안 AI 채점 → 루브릭 기반 점수 산출 → 유사 답안 그룹화
- **입시 시즌**: 학생별 합격 시뮬레이션 → 교과 이수 설계 → 진로 로드맵 상담

## 4. 핵심 기능 상세

### 4.1 입시 네비게이터

> **목적**: 공공데이터 기반으로 학생의 합격 가능성을 예측하고, 목표 대학에서 역산한 교과 이수 전략과 진로 로드맵을 제공

#### 4.1.1 합격 시뮬레이션
- **입력**: 학생 프로필 (내신 등급, 모의고사 백분위, 희망 전공)
- **출력**: 대학·학과별 합격 확률 (%) + 안정/적정/소신 분류
- **데이터 소스**: 어디가(adiga.kr), 대학알리미 등 공공데이터 API (197개 대학)
- **표시 정보**: 합격 확률 프로그레스바, 70% 컷 등급 기준

#### 4.1.2 교과 이수 설계
- 목표 대학의 가산점·우대 과목 기반으로 이수 과목 추천
- **시나리오 비교**: 과목 변경 시 합격률 변동 시뮬레이션 (예: "미적분II 추가 이수 시 KAIST +8%")
- 현재 이수 현황 시각화 (완료/이수중/미이수)
- AI 추천 및 경고 (예: "물리학II 미이수 시 62% → 41% 하락")

#### 4.1.3 진로 로드맵
- 현재 학년부터 목표 입학까지의 단계별 타임라인 시각화
- 각 단계별 권장 활동 (이수 과목, R&E, 수시 전략 등)

#### 4.1.4 필요 데이터 상세

| | 이수 과목 목록 | 공통/일반선택/진로선택/융합선택 구분 포함 |
| | 비교과 활동 | 동아리, R&E, 자기평가서 등 (자기소개서는 폐지됨) |
| **대학 측** | 전형별 모집인원 | 교과/종합/논술/실기 유형별 |
| | 전형별 경쟁률 | 최근 3~5년 |
| | 합격자 등급 분포 | 70%컷, 50%컷, 평균 — **합격 예측의 핵심 데이터** |
| | 전형 요소 반영비율 | 학생부 %, 면접 %, 논술 % 등 |
| | 교과 반영 방법 | 반영 교과, 반영 과목 수, 가산점 과목 |

| | 권장/필수 이수과목 | 학과별 (2028 이후 대학이 의무 공시) |
| **통계** | 충원율 | 추가합격 비율 — 실질 경쟁률 산정에 필수 |
| | 이월인원 | 수시 미충원 → 정시 이월 규모 |

#### 4.1.6 예측 모델

##### 합격 확률 예측

**MVP — 통계 기반 시뮬레이션 (ML 불필요)**
1. 어디가 입결 데이터의 합격자 등급 분포에 Beta 분포를 피팅
2. 해당 분포에서 학생의 등급 위치를 계산하여 확률 산출
3. 보정 변수 적용: 수능최저 충족률, 경쟁률 추세(3년), 모집인원 변동

**고도화 — 지도학습 모델 (상업 데이터 확보 시)**
- 알고리즘: XGBoost / LightGBM (소규모 테이블 데이터에 최적)
- 입력: (학생 프로필, 대학·학과, 전형유형)
- 출력: 합격/불합격 이진 분류 → Platt scaling으로 확률 보정
- 훈련 데이터: 진학사 등 상업 데이터 (~20만 수험생/년)

##### 교과 이수 추천

ML 불필요 — **규칙 기반 제약 최적화**
1. 과목별 한계효용 점수 계산: 이수 시 각 목표 대학 합격률 변동량
2. 제약 조건: 잔여 학점, 학교 개설 과목, 선수과목 체인
3. 탐욕적 최적화 또는 정수 선형 계획법(ILP)으로 최적 과목 조합 도출
4. What-if 시나리오: 개별 과목 토글 시 합격률 변동 시뮬레이션

#### 4.1.7 데이터 파이프라인

```
[외부 소스]
  ├─ API 수집기 (야간 배치) ────── 대학알리미, 공공데이터포털
  ├─ 스크래퍼/파서 (연 1~2회) ──── 어디가, 모집요강 PDF, 수능 분포표
  └─ 수동/상업 데이터 임포트 ───── CSV/Excel

         ↓

[정규화 레이어]
  ├─ 대학/학과 ID 매핑 (소스별 식별자 통합)
  ├─ 학과명 정규화 ("물리천문학부" = "물리학과" 등 별칭 관리)
  ├─ 등급 체계 변환 (9등급 ↔ 5등급 성취평가제)
  └─ 전형 유형 분류 통일

         ↓

[데이터 저장소] ── PostgreSQL + Redis 캐시
  ├─ universities / departments
  ├─ admission_results (입결: 경쟁률, 70%컷, 50%컷, 평균)
  ├─ admission_requirements (전형요소, 반영비율, 수능최저)
  ├─ course_catalog (2022 개정 교육과정 과목 체계)
  ├─ recommended_courses (대학별 권장 이수과목)
  └─ csat_distributions (수능 점수 분포표)

         ↓

[예측 엔진] ── 합격 확률 / 이수 추천
         ↓
[백엔드 API] → [프론트엔드]
```

#### 4.1.8 2028 대입 개편 — 데이터 영향

| 변경사항 | 영향 | 대응 |
|---------|------|------|
| 9등급 → 5등급 성취평가제 | 기존 입결 데이터와 직접 비교 불가 | 등급 변환 함수 구축 + 2029년 신규 입결 데이터로 모델 재훈련 |
| 수능 선택과목 폐지 | 정시 환산점수 체계 전면 변경 | 신규 수능 분포표 기반 별도 모델 구축 |
| 고교학점제 전면 시행 | 과목 경로 다양화 → 이수 설계 복잡도 증가 | 과목 DB 확장 + 학교별 개설 과목 데이터 수집 |
| 대학 권장과목 의무 공시 | 이수 설계의 핵심 데이터가 공개됨 | Tier B → Tier A 격상, 공시 데이터 자동 수집 |

> **콜드 스타트 문제**: 2028 첫 입시에는 신체제 입결 데이터가 0건. 2029년 3월까지는 휴리스틱 등급 매핑으로 운영 필요.

---

### 4.2 생기부 AI

> **목적**: 교사의 관찰 기록을 바탕으로 세특·창체·행발을 AI가 자동 합성하고, 품질 검증까지 한 화면에서 처리

#### 4.2.1 관찰 기록 시스템
- **인라인 메모 입력**: 수업 중 관찰 내용을 태그(학업역량/공동체/자기주도/진로탐색)와 함께 빠르게 기록
- **AI 자동 태깅**: 메모 내용에서 적절한 태그를 자동 추천
- **관찰 타임라인**: 시간순 정렬된 관찰 기록을 색상 코딩으로 시각화

#### 4.2.2 세특 (세부능력 및 특기사항) 자동 합성
- 축적된 관찰 메모를 **기-승-전-결 구조**로 자동 합성
- **바이트 카운터**: 1,500 Byte 제한 실시간 체크 (프로그레스바 + 초과 시 빨간색 경고)
- **성취기준 자동 매핑**: 작성된 세특에 해당하는 교육과정 성취기준 코드 자동 태깅 (예: [10물리01-03])
- **AI 역방향 세특 가이드**: 목표 대학 기반으로 세특에 강조해야 할 역량 방향 제시 (예: "서울대 물리천문학부 → 실험설계 역량, 수학적 모델링, 논문 독해력, 협업·리더십")

#### 4.2.3 창체 (창의적 체험활동) 자동 합성
- **하위 탭**: 자율활동 / 동아리 / 진로활동
- 각 영역별 500자 기준 바이트 카운터
- AI 기반 자동 작성 + 재생성 기능

#### 4.2.4 행발 (행동특성 및 종합의견) 자동 합성
- 담임 총평 형식의 자동 합성
- 학년 스토리라인 일관성 검사

#### 4.2.5 품질 검증 엔진
| 검증 항목 | 설명 |
|-----------|------|
| **유사도 검사** | 학급 내 학생 간 세특 유사도 % 표시 + 겹치는 표현 하이라이트 (예: "김민수-박지영 87% 유사") |
| **금지어 검출** | 교육부 가이드라인 위반 용어 자동 감지 (예: "교외 수상") |
| **바이트 제한** | 1,500 Byte 실시간 체크 + 초과 시 시각적 경고 |
| **차별화 제안** | 유사도 경고 시 "차별화 제안" 버튼으로 대체 표현 AI 생성 |

#### 4.2.6 출력
- **NEIS 복사** 버튼: 클립보드에 바로 복사하여 NEIS 시스템에 붙여넣기 가능
- **수정** 모드: 교사가 직접 텍스트 편집 가능

### 4.3 채점 가드

> **목적**: 서논술형 답안을 루브릭 기반으로 AI가 채점하고, 근거 문장 매칭과 유사 답안 그룹화를 통해 공정한 채점 지원

#### 4.3.1 시험 관리
- **시험 선택**: 드롭다운으로 시험 종류 전환 (예: "2학기 중간고사 물리I 서술형", "뉴턴 역학 심화 탐구 보고서")
- **학생 목록**: 좌측 패널에서 학생별 답안 빠른 전환

#### 4.3.2 답안 분석
- **근거 문장 하이라이트**: 답안 내 채점 근거가 되는 문장을 색상별로 구분 표시
  - 파란색: 개념 이해
  - 초록색: 실험 설계
  - 보라색: 비판적 사고
  - 노란색: 논리 전개
  - 주황색: 성찰·확장
- **근거 매칭 태그**: 답안 하단에 감지된 역량 태그 표시

#### 4.3.3 AI 채점 결과
- **총점 표시**: 대형 점수 카드 (예: 17/20)
- **루브릭 항목별 점수**: 5개 채점 기준별 점수 + 감점 사유 상세 표시

| 채점 기준 | 배점 | 설명 |
|-----------|------|------|
| 개념 이해 | /5 | 물리 법칙의 정확한 정의 및 표현 |
| 실험 설계 | /5 | 변인 설정, 비례 관계 검증 |
| 비판적 사고 | /5 | 오차 분석, 마찰 계수 역추산 등 |
| 논리 전개 | /5 | 인과 관계의 충분성 |
| 성찰·확장 | /5 | 후속 탐구 방향 제시 |

#### 4.3.4 유사 답안 그룹화
- 학생 간 답안 유사도 자동 감지 (예: "82% 유사")
- 유사 답안 학생 목록 표시 (예: 김민수, 이준호, 박서준)

#### 4.3.5 출력
- **확정** 버튼: 채점 결과 확정
- **수정** 버튼: 교사가 점수/사유 직접 수정
